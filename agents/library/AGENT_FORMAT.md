# Agent 定义格式规范

> 从 agency-agents（172 个模板）中提取的最佳实践模式

## YAML Frontmatter（必须）

```yaml
---
name: Agent 名称
description: 一句话描述（英文，用于自动匹配）
color: "#HEX"     # 可选，终端/看板中的标识色
emoji: 🔬         # 可选，快速识别符号
vibe: 一句话人设   # 可选，Agent 的"语气定位"
---
```

## Markdown 正文结构

### 1. 角色声明（第一段）

> "You are **{角色名}**, ..."

用一段话说清：你是谁、你的核心能力、你的行事风格。

### 2. 身份与记忆 `## 🧠 Your Identity & Memory`

| 字段 | 说明 |
|------|------|
| **Role** | 角色定位（一句话） |
| **Personality** | 行事风格 |
| **Memory** | 你记住哪些模式（领域经验） |
| **Experience** | 你见过哪些场景 |

### 3. 核心使命 `## 🎯 Your Core Mission`

编号的任务清单，每个任务下列出具体动作：

```markdown
### 1. 任务名
- 具体动作 1
- 具体动作 2
- **默认要求**: 每次必须做的事
```

### 4. 关键规则 `## ⚖️ Rules`

用 ✅ / ❌ 格式：

```markdown
✅ 必须建立量化评估标准（"准确率 > 98%"而非"准确率要高"）
❌ 禁止跳过数据现状调研
❌ 禁止在没有基线对比的情况下给出技术推荐
```

### 5. 工作流 `## 🔄 Workflow`

编号的决策顺序：

```markdown
1. 收集需求 → 2. 分析数据 → 3. 设计方案 → 4. 执行验证 → 5. 输出报告
```

### 6. 输出物 `## 📦 Deliverables`

明确列出 Agent 完成工作后产出的文件和格式。

### 7. 成功指标 `## 📊 Success Metrics`

用具体数字：

```markdown
- 需求覆盖率 > 95%（无遗漏的关键需求）
- 方案可量化评估（每个推荐都有数据支撑）
```

### 8. 记忆域 `## 💾 Memory Domains`

Agent 应该记住的领域知识模式：

```markdown
记住：
- 历史项目的技术选型教训
- 数据质量问题的常见模式
- 评估指标与业务目标的映射关系
```

## 从 agency-agents 学到的关键模式

| 模式 | 做法 | 示例 |
|------|------|------|
| 规则用 ✅❌ | 视觉一目了然 | "✅ 必须建数学模型 ❌ 禁止拍脑袋估产能" |
| 代码示例是领域核心 | 不是通用编程，是领域逻辑 | CP-SAT 排产代码，不是 Hello World |
| 成功指标是数字 | 可验证 | "makespan 偏差 <5%，求解时间 <60s" |
| 工作流是决策序列 | 不是功能列表 | "1.分类→2.建模→3.选solver→4.benchmark" |
| 记忆域明确命名 | 告诉 Agent 该记什么 | "记住：换线时间规律、产能波动" |
| 人设有态度 | 不是中立的，有鲜明风格 | "Skeptical but collaborative" |
